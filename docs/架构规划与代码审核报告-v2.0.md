**当前状态：** 已有 8 个 ORM 模型（User, AgentProfile, Requirement, RequirementQuote, Order, Review, CreditRecord + BaseModel 基类），使用统一 models/models.py 文件。

**发现的问题及建议：**

| # | 问题 | 建议 |
|---|------|------|
| 1 | models/ 下有重复定义（models.py + user.py 都定义了 User），两个版本不一致 | 统一使用 models.py，删除 user.py / agent.py / demand.py / order.py（旧版） |
| 2 | AgentProfile 表缺少 `description_vec` 字段 — ai_service.py 的 match_agents 引用了 `ap.description_vec`，但模型里没有 | 需要在 AgentProfile 增加 `description_vec = Column(Vector(1536))` 字段 |

### 2.2 🔴 P0 — 必须立即修复

#### P0-1: config.py 缺少关键配置项 — 导致运行时崩溃

**位置：** `app/config.py`、`app/services/ai_service.py`（行54, 65-66）、`app/api/v1/agents.py`（行315）、`app/api/v1/orders.py`（行299）

**问题描述：** 代码中引用了多个 config 中不存在的字段：
- `settings.QWEN_API_KEY` — ai_service.py 使用，config 中定义是 `dashscope_api_key`
- `settings.QWEN_MODEL` — ai_service.py 使用，config 中定义是 `ai_model`
- `settings.PLATFORM_FEE_RATE` — agents.py 使用，config 中不存在
- `settings.MAX_MODIFY_TIMES` — orders.py 使用，config 中不存在
- `settings.WECHAT_APP_ID` / `settings.WECHAT_APP_SECRET` — auth.py 使用，config 中已定义 ✅

**影响：** 运行时 `AttributeError`，功能完全不可用。

**修复方案：**
```python
# config.py 补充：
platform_fee_rate: float = 0.10            # 平台费率 10%
max_modify_times: int = 3                   # 最大修改次数
qwen_api_key: str = ""                      # 通义千问 API Key
qwen_model: str = "qwen-plus"              # 通义千问模型名

# 属性别名（兼容 DashScope SDK 使用 dashscope_api_key）：
@property
def QWEN_API_KEY(self) -> str:
    return self.qwen_api_key or self.dashscope_api_key

@property
def QWEN_MODEL(self) -> str:
    return self.qwen_model or self.ai_model

@property
def PLATFORM_FEE_RATE(self) -> float:
    return self.platform_fee_rate

@property
def MAX_MODIFY_TIMES(self) -> int:
    return self.max_modify_times
```

#### P0-2: AgentProfile 缺少 description_vec 字段 — 匹配引擎查询崩溃

**位置：** `app/services/ai_service.py` 行150-163（match_agents）、`app/models/models.py` AgentProfile 类

**问题描述：** ai_service.py 的 match_agents 函数查询了 `ap.description_vec` 字段来做向量相似度匹配，但 AgentProfile 模型中**没有定义这个字段**，SQL 查询会直接崩溃（列不存在）。

**修复方案：**
```python
# models/models.py AgentProfile 增加：
description_vec = Column(Vector(1536), nullable=True, comment="Agent能力描述向量")
```

#### P0-3: admin.py 中混合使用了 ORM 查询和原始 SQL 表查询

**位置：** `app/api/v1/admin.py` 行52、64

**问题描述：** admin 模块直接使用 `User.__table__.select()` 原始表查询，而不是 SQLAlchemy ORM select() 语句。这在 async session 下行为不一致：

```python
# 当前代码（不可靠）：
result = await db.execute(User.__table__.select().order_by(...).offset(offset).limit(page_size))

# 应改为（正确）：
from sqlalchemy import select
result = await db.execute(select(User).order_by(User.created_at.desc()).offset(offset).limit(page_size))
```

**影响：** 管理后台的 users/orders/requirements 列表查询在 async 环境下可能返回不一致结果。

### 2.3 🟡 P1 — 强烈建议修复

#### P1-1: auth.py 验证码存储在内存 — 重启丢失

**位置：** `app/api/v1/auth.py` 行217

**问题：** `_phone_verify_codes: dict[str, str] = {}` 是 Python 进程内存字典，多 worker 或重启后验证码丢失。已有注释 TODO Redis，应尽快实现。

#### P1-2: requirements.py 的列表接口无认证但取回所有需求

**位置：** `app/api/v1/requirements.py` 行127-163

**问题：** `GET /requirements/` 是公开接口，不需要认证，但对所有已登录用户都可见所有需求的状态，包括草稿和已取消的。建议默认只返回 `status='open'` 的需求。

#### P1-3: requirements.py 中 _to_response 函数未被使用

**位置：** `app/api/v1/requirements.py` 行31-46

**问题：** `_to_response()` 函数定义了但没被实际调用（实际用的是 `RequirementResponse.model_validate()`），应删除。

#### P1-4: admin.py 缺少 middleware 级别的权限验证

**位置：** `app/api/v1/admin.py`

**问题：** 每个 admin 端点都需要手动检查 `if current_user.role != "admin"`，代码重复。建议抽一个 `get_current_admin_user` 依赖：

```python
async def get_current_admin_user(
    current_user: User = Depends(get_current_active_user),
) -> User:
    if current_user.role != "admin":
        raise HTTPException(status_code=403, detail="需要管理员权限")
    return current_user
```

#### P1-5: ai_service.py 的 SQL 注入风险

**位置：** `app/services/ai_service.py` 行150

**问题：** `match_agents` 使用 Python 字符串拼接构建 pgvector 查询：
```python
f"CAST('{embedding_literal}' AS vector(1536))"
```
虽然 embedding 是内部生成的浮点数数组而非用户输入，但这仍是**反模式**。建议使用 SQLAlchemy 的 bind 参数：

```python
query = text("""
    SELECT ... 
    WHERE ... ORDER BY description_vec <=> CAST(:embedding AS vector(1536))
""")
result = await db.execute(query, {"embedding": embedding_literal, "limit": limit})
```

#### P1-6: orders.py 中 `reject_order` 的 `reason` 参数使用 query param

**位置：** `app/api/v1/orders.py` 行275

**问题：** `reject_order` 把 `reason` 定义为函数参数而非 Pydantic model 或 request body。Query 传参对于中文长文本会出现编码问题。
