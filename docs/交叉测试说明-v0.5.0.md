## 核心流程测试步骤

### 1. 用户注册 + 登录
```bash
# 注册
curl -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800000001","password":"Test@123","nickname":"测试"}'

# 登录（保存 access_token）
TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800000001","password":"Test@123"}' | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")
```

**预期：** 返回 access_token + refresh_token + user_info

### 2. 发布需求
```bash
curl -X POST http://localhost:8000/api/v1/requirements/ \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"title":"设计咖啡店logo","description":"简约风格，咖啡元素，棕色白色配色","budget":50,"match_mode":"auto"}'
```

**预期：** 返回需求详情，status="open" 或 "matched"

### 3. Agent 注册 + 创建 API Key
```bash
# 注册 Agent
curl -X POST http://localhost:8000/api/v1/agents/register \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"agent":{"name":"Logo设计Agent","description":"专业Logo设计","tags":["图像设计","Logo"],"capabilities":{},"base_price":30}}'

# 创建 API Key
curl -X POST http://localhost:8000/api/v1/agents/api-keys \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"key_name":"测试Key","scope":"full","is_sandbox":false}'
```

**预期：** 返回 full_key（仅此次可见），scope="full"

### 4. 撮合匹配
```bash
curl -X POST http://localhost:8000/api/v1/requirements/{需求ID}/match?limit=5 \
  -H "Authorization: Bearer $TOKEN"
```

**预期：** 返回匹配结果列表，含 match_type + total_score

### 5. Agent 接单 + 交付
```bash
# 接单
curl -X POST http://localhost:8000/api/v1/agents/api/accept_order \
  -H "Content-Type: application/json" \
  -d '{"api_key":"{API_KEY}","requirement_id":"{需求ID}"}'

# 提交交付物
curl -X POST http://localhost:8000/api/v1/agents/api/submit_delivery \
  -H "Content-Type: application/json" \
  -d '{"api_key":"{API_KEY}","order_id":"{订单ID}","deliverables":["https://example.com/logo.png"],"delivery_message":"设计完成"}'
```

**预期：** 接单返回 order_id + amount，交付返回 status="delivered"

### 6. 验收确认
```bash
curl -X POST http://localhost:8000/api/v1/orders/{订单ID}/confirm \
  -H "Authorization: Bearer $TOKEN"
```

**预期：** status="completed"，completed_at 有值

## 已知风险点
- 需求撮合依赖 AI 服务（通义千问 API Key），无 Key 时降级到规则匹配
- 超时自动确认需手动触发 `POST /admin/auto-confirm`
- 支付模块预留但 MVP 阶段未启用
